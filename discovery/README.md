# Discovery

Discovery is where ADX learns.

Everything in discovery is evidence or a hypothesis.

Nothing becomes part of the platform until it has been observed,
challenged, and validated.

---

## The ADX Development Process

```
Stage 1 — Observe
Go to the business.
Watch.
Ask.
Take notes.

    ↓

Stage 2 — Discover
Find operations.
Find decisions.
Find concepts.
Find rules.

    ↓

Stage 3 — Validate
Observe different scenarios.
Break your assumptions.
Refine the model.

    ↓

Stage 4 — Document
Only after repeated validation.

    ↓

Stage 5 — Design
UI
APIs
Modules
Events

    ↓

Stage 6 — Build
```

---

## How discovery relates to documentation and code

```
discovery/    →  Evidence and hypotheses
docs/         →  Understanding and decisions
code/         →  Implementation
```

Discovery captures what you've observed.

Documentation captures what you understand.

Code captures what you've decided.

Discovery comes first.

---

## Rules

1. **Evidence before abstraction.** Every abstraction should be traceable to repeated observations.
2. **Observe before you design.** Never start with entities or APIs.
3. **Validate before you document.** One observation is a hypothesis. Multiple observations are evidence.
4. **Question everything.** If you can't point to an observation, you probably invented it.
5. **When the model changes.** The current model should change only when repeated evidence cannot be explained by it. Do not change the model because a new idea seems better. Change it because the evidence demands it.
6. **Separate evidence from interpretation.** Record what happened before explaining why it happened. Different interpretations may arise from the same evidence.

## Methodology Status

Version: Discovery Method v1

Status: Frozen

The discovery methodology should change only when repeated evidence
shows that it fails to produce an accurate understanding of a business.

Do not change the methodology because a different approach appears
more elegant.

## Idea Lifecycle

Every idea follows a path:

```
Idea
    ↓
Hypothesis
    ↓
Evidence
    ↓
Validated Understanding
    ↓
Documentation
    ↓
Principle
```

Most ideas die.

Some become hypotheses.

A few become evidence.

Validated understandings become documentation.

Very few become principles.

Principles should be rare.

Documentation can grow much more freely.
